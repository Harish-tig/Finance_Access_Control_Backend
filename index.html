Finance Dashboard API

A role-based access control backend for managing financial records, dashboards, and user accounts. Built with FastAPI, SQLAlchemy, Redis, and JWT authentication.

🔐

JWT Authentication

Access + refresh token flow with configurable expiry.

👥

Role-Based Access

Three roles — Viewer, Analyst, Admin — with hierarchical permissions.

📊

Dashboard Analytics

Income, expenses, category totals, and monthly trends.

⚡

Redis Caching

Dashboard results cached with auto-invalidation on writes.

🔑 Authentication

POST /api/v1/auth/login Public ▶

Authenticate with email and password. Returns access + refresh tokens.

Request Body

{
  "email": "admin@example.com",
  "password": "Admin@1234"
}

Response 200

{
  "access_token": "eyJhbGci...",
  "refresh_token": "eyJhbGci...",
  "token_type": "bearer"
}

Error 401

{ "detail": "Incorrect email or password." }
POST /api/v1/auth/refresh Public ▶

Exchange a valid refresh token for a new access + refresh token pair.

Request Body

{
  "refresh_token": "eyJhbGci..."
}

Response 200

{
  "access_token": "eyJhbGci...",
  "refresh_token": "eyJhbGci...",
  "token_type": "bearer"
}

🔄 Authentication Flow

1

Login

POST /api/v1/auth/login with email + password. Receive an access token (30 min) and a refresh token (7 days).

2

Use Access Token

Include in the Authorization header: Bearer <access_token>

3

Refresh

When the access token expires, POST /api/v1/auth/refresh with the refresh token to get a new pair.

🚀 Project Setup

1

Clone the repository

git clone https://github.com/Harish-tig/Finance_Access_Control_Backend.git
cd Finance_Access_Control_Backend
2

Create virtual environment

python3 -m venv .venv
source .venv/bin/activate
3

Install dependencies

pip install -r requirements.txt
4

Configure environment

Edit the .env file in the project root. At minimum, change SECRET_KEY to a strong random value for production.

SECRET_KEY=your-production-secret-here
5

Start Redis (optional)

If you want caching and rate limiting. The app works without Redis (graceful fallback).

redis-server
6

Run the server

uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

On first run, the database and default admin user are created automatically.
