Finance Dashboard
API Reference

A role-based access control backend for managing financial records, dashboards, and user accounts. Built with FastAPI, SQLAlchemy, Redis, and JWT authentication.

Authentication

POST /api/v1/auth/login Public ▶

Authenticate with email and password. Returns access and refresh tokens.

Request Body

{
  "email": "admin@example.com",
  "password": "Admin@1234"
}

Response 200

{
  "access_token": "eyJhbGci...",
  "refresh_token": "eyJhbGci...",
  "token_type": "bearer"
}

Error 401

{ "detail": "Incorrect email or password." }
POST /api/v1/auth/refresh Public ▶

Exchange a valid refresh token for a new access and refresh token pair.

Request Body

{
  "refresh_token": "eyJhbGci..."
}

Response 200

{
  "access_token": "eyJhbGci...",
  "refresh_token": "eyJhbGci...",
  "token_type": "bearer"
}

Authentication Flow

1

Login

POST to /api/v1/auth/login with your email and password. The response contains an access token valid for 30 minutes and a refresh token valid for 7 days.

2

Use the access token

Include the token in every protected request via the Authorization header.

Authorization: Bearer <access_token>
3

Refresh when expired

When the access token expires, POST to /api/v1/auth/refresh with the refresh token body. A new token pair is returned without re-entering credentials.

Project Setup

1

Clone the repository

git clone https://github.com/Harish-tig/Finance_Access_Control_Backend.git
cd Finance_Access_Control_Backend
2

Create a virtual environment

python3 -m venv .venv
source .venv/bin/activate
3

Install dependencies

pip install -r requirements.txt
4

Configure environment

Edit the .env file in the project root. At minimum, replace SECRET_KEY with a strong random value before deploying to production.

SECRET_KEY=your-production-secret-here
5

Start Redis (optional)

Enables caching and rate limiting. The application runs without Redis with a graceful fallback.

redis-server
6

Run the server

uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

On first run, the database schema and default admin account are created automatically.
